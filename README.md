# Sberoom

Внутренний сервис для управления долей удалённой работы в подразделении.
Цель — сделать целевой показатель **70 / 30 (офис / удалёнка)** прозрачным
для всех сотрудников, не вводя ни блокировок, ни согласований.

> MVP по [техническому заданию](#тз). Mobile-first, полноценно работает и
> на десктопе.

## Что внутри

- 📊 **Сводка** — индикатор удалёнки за месяц с цветовыми зонами
  (🟢 ≤30 %, 🟡 30–35 %, 🟠 35–40 %, 🔴 >40 %), блок «Сегодня»
  (офис / удалённо / доля), KPI за месяц, дельта к прошлому месяцу,
  прогноз итогов месяца.
- 📅 **Календарь** — выбор «Офис / Удалённо» на сегодня и до трёх
  рабочих дней вперёд. При выборе удалёнки — модалка с текущей и
  прогнозируемой долей, ситуацией по команде, необязательной причиной
  и случайным дружелюбным сообщением.
- 👥 **Команды** — сводка по подразделению и рейтинг команд по доле
  присутствия в офисе.
- 👤 **Я** — личная статистика за неделю и месяц + история отметок.
- Без блокировок: даже при превышении цели выбор удалёнки доступен,
  меняется только цвет индикатора и появляется предупреждение.

## Запуск

Достаточно любого статического веб-сервера в корне репозитория.

### Вариант 1 — Node (рекомендуется, без зависимостей)

```bash
npm start
# → Sberoom доступен на http://localhost:5173/
```

`npm start` запускает `server.js` — крошечный HTTP-сервер на стандартной
библиотеке Node, никаких внешних пакетов ставить не нужно.

### Вариант 2 — Python

```bash
python3 -m http.server 5173
# → http://localhost:5173/
```

### Вариант 3 — открыть напрямую

Можно просто открыть `index.html` в браузере, но некоторые браузеры
ограничивают `localStorage` для `file://`, поэтому варианты 1–2 надёжнее.

## Структура

```
.
├── index.html      # разметка, навигация, табы
├── styles.css      # стили: мобильный + десктопный layout, цветовые зоны
├── data.js         # моковые данные: 80 сотрудников × 5 команд + история
├── app.js          # логика: расчёт KPI, рендер вкладок, модалка, persistence
├── server.js       # zero-dep HTTP-сервер для локального запуска
├── favicon.svg
└── package.json    # только npm-скрипт start
```

### Где менять данные

- **Список сотрудников и команд** — `data.js`, константы `TEAMS`,
  `TEAM_SIZE`, `EMPLOYEES` (генерируются детерминированно).
  В реальной системе сюда поедет импорт из Excel / HR-системы.
- **Текущая дата** — `data.js`, константа `TODAY` (`YYYY-MM-DD`).
- **Целевой показатель** — `TARGET_REMOTE_SHARE` (по умолчанию `0.30`).
- **Максимальный горизонт планирования** — `MAX_FORWARD_DAYS` (по умолчанию `3`).
- **Цветовые зоны индикатора** — функция `zone()` в `app.js`.
- **Дружелюбные сообщения** — `GAMIFICATION_LINES` в `data.js`.

Личные отметки текущего пользователя сохраняются в `localStorage`
(ключ `sberoom_user_v1`). Очистить — DevTools → Application → Local Storage.

## Что в MVP не входит (намеренно)

- Авторизация, роли, согласования.
- Карта офиса и бронирование мест.
- Бэкенд и база данных (демо-стенд статический).
- Обмен днями между сотрудниками — фича отмечена в ТЗ как «будущая версия».

## ТЗ

Подробное техническое задание — в описании Pull Request и в исходном брифе
подразделения. Ключевые принципы, которые MVP сохраняет:

1. Простой интерфейс, никаких ограничений и согласований.
2. Все видят одинаковую информацию.
3. Основной фокус — метрика удалённой работы, а не контроль сотрудников.
4. Mobile-first.
